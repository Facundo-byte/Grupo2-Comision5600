# Nomenclatura del Proyecto
Nuestro equipo ha adoptado las siguientes convenciones de nomenclatura para garantizar la consistencia, legibilidad y el cumplimiento de las buenas prácticas en la base de datos (SQL Server) y el código.
**Tablas (Entidades)**: `camelCase` Siempre en **singular**.
**Store Procedures (SP)**: `snake_case` (minúsculas y guion bajo) Descriptivo de la acción a realizar y prefijo SP.
**Columnas (Atributos)**: `snake_case` (minúsculas y guion bajo) Descriptivo.
**Índices**: `snake_case` (minúsculas y guion bajo) Descriptivo y prefijo IX con continuacion de tabla y campos incluidos.
